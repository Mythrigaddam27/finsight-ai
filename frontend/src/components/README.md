# Components

Reusable React components organized by feature domain.

## Structure

- **layout**: Application shell, navigation, and global layout components
- **dashboard**: Dashboard widgets, cards, and analytics visualization components
- **expenses**: Expense-related UI components (forms, lists, tables, modals)

## Guidelines

- Components should be functional components (React hooks)
- Keep components small, focused, and single-responsibility
- Use composition over prop drilling
- Export components from index.ts for clean imports
- Document component props with TypeScript interfaces
